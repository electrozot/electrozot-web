HOW IT WORKS:
================================================================================

STEP 1: Setup (One-time)
   → Run: admin/simple-skill-matching.sql
   → Adds t_skills column to technician table

STEP 2: Add Technician Skills
   → Admin opens: admin-add-technician-with-skills.php
   → Ticks services technician can handle
   → Example: Tick "AC (Split) - Repair", "Refrigerator - Repair"
   → Skills saved as: "AC (Split) - Repair,Refrigerator - Repair"

STEP 3: Customer Books Service
   → Customer selects: "AC (Split) - Repair"
   → Booking created with service name

STEP 4: Admin Assigns Booking
   → Admin opens: admin-assign-booking-skill-match.php?booking_id=123
   → System shows: "Required Skill: AC (Split) - Repair"
   → **Only technicians with this skill are displayed**
   → Admin clicks "Assign" button
   → Done!

================================================================================
THE MATCHING LOGIC:
================================================================================

SQL Query:
```sql
SELECT * FROM tms_technician 
WHERE FIND_IN_SET('AC (Split) - Repair', t_skills) > 0
  AND t_status = 'Available'
  AND t_current_bookings < t_booking_limit
```

Example:
- Booking needs: "AC (Split) - Repair"
- Technician A has: "AC (Split) - Repair,Refrigerator - Repair" ✅ SHOWN
- Technician B has: "Washing Machine - Repair,TV Repair" ❌ HIDDEN
- Technician C has: "AC (Window) - Repair" ❌ HIDDEN (different service)
